Fries Utils Pack 总览

Fries Utils Pack 是一个针对 Unity 2D/3D 的常用工具箱，
它通过 Git URL 在 Unity Package Manager 中加载。可以通过 Ctrl + F6 拉取线上更新
国内访问需要通过 Git Bash 设置在 Gitee 上的镜像仓库转发来拉取

内容

Custom Data

一个 GameObject 的额外自定义数据类，支持在 Editor 和 Play Mode 中给 GameObject 动态的附加任意数量的特定类型的数据
这些数据在拥有 GameObject 的情况下，可以全局 O(1) 访问和写入

Multi Tag

在 O(1) 的访问中，支持 一个 GameObject 可以有无限数量的 Tag

Event Functions

这是一个模拟 Unity Event Function 的系统，它可以让任意 EventFunctionSubscriber 拥有 FixedUpdate, Update, LateUpdate 事件

Insertion Event System

支持在任意类声明事件，在任意代码行 Trigger 事件，并且在任意地点声明监听器

Go Buttons

这是一个基于 Raycaster 和 IPointer 两种模式的 按钮系统，它可以将任何带有碰撞箱的 GameObject / 任意能拦截到点击事件的 UI 变成一个按钮

Hierarchy Button

这是一个可以在 Unity Hierarchy 窗口中快速定义按钮和点击后执行的代码的功能

Static Method Field

这是一个可以将 特定脚本中 的 Static 方法序列化并显示在 Unity Inspector 中的模块

Task Performer

这个类允许调用者在任何地方延迟执行，周期性重复执行，条件满足时执行，条件满足后周期性重复执行某段代码
也支持从这个类运行 .exe 程序。除了执行 exe 外，其他操作支持查询运行状态和开始前取消其执行，也支持传参与记录参数

Pool

这个功能允许快捷的创建各种包括 MonoBehaviour 的对象池

Component Cache

这个类缓存 GetComponent 的结果并在之后的调用中返回之前的实例。在查询某个 GameObject 上的单一特定 MonoBehaviour 时有 O(1) 的效率，
但是如果未命中会每次产生 GetComponent 原先的时间复杂度。可以结合 Multi Tag 避免查询不存在的 MonoBehaviour
这个类对 GetComponents 的支持不好，也不建议使用在可能会 动态 Remove / AddComponent 的情境下使用

Prefab Parameter

这个类允许你在生成 Prefab 时向 Awake 和 Start 方法中传递参数
