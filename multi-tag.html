Multi Tag

MultiTag 是一种 MonoBehaviour ，它为 GameObject 提供“多标签（multi-tag）”能力，并通过一组扩展方法在运行时增、删、查标签。
它使用一个 静态 的、以 GameObject 为键的映射结构缓存标签集合，从而实现 O(1) 级别的常见操作开销（摊销）。

快速开始

  1. 在需要支持多标签的对象上 添加组件： MultiTag。

  2. 在 Inspector 的 Tags 列表中录入初始标签（如： Enemy, Flying, Boss）。

  3. 运行时使用扩展方法进行操作：

using Fries; public class SpawnedEnemy : MonoBehaviour { void Start() { // 如果在 Inspector 中没有添加标签，也可以这样在代码里添加 gameObject.addTag("Enemy"); gameObject.addTag("Flying"); // 查询标签 if (gameObject.hasTag("Enemy")) { // ... } // 移除标签 gameObject.removeTag("Flying"); } }

Awake() 会把 Inspector 中的 tags 写入到 tagData ，并将一条说明性文本插入到 tags[0]：
"Tags are not editable in inspector during runtime"。 请不要把 tags[0] 当作有效标签。

API 参考

扩展： Fries.MultiTagExts

方法

签名

行为

addTag

void addTag(this GameObject, string tag)

若不存在条目则创建，随后添加标签。是否去重取决于 DictList。

removeTag

void removeTag(this GameObject, string tag)

在存在条目与标签时移除。

hasTag

bool hasTag(this GameObject, string tag)

判断目标对象是否拥有指定标签。

线程安全

  • Unity 的 API（GameObject 等）仅在主线程安全。

  • 当前实现没有并发保护； 请只在主线程操作标签。

故障排查

  • 症状： hasTag 总是返回 false

    • 排查 ：确认是否已挂载 MultiTag 组件；检查 Awake 是否被调用（对象未激活时不会调用）。

    • 排查 ：确认 addTag 已经被调用
